# 📱 두리무역 웹사이트 모바일 최적화 설명서

## 🎯 **프로젝트 개요**
- **목표**: 현재 "설명서형" 웹사이트를 모바일 친화적인 "컴팩트형"으로 전환
- **핵심 문제**: 모바일에서 너무 크고 길게 표시되어 사용자 피로도 증가
- **해결 방향**: 정보 밀도 증가 + 인터랙션 개선 + 성능 최적화

---

## 📊 **현재 상태 분석**

### **문제점**
1. **히어로 섹션**: `h-screen` (100vh)로 모바일에서 과도하게 큼
2. **폰트 크기**: `text-7xl` (96px)로 모바일에서 읽기 어려움  
3. **섹션 간격**: `py-24` (96px)로 과도한 여백
4. **정보 밀도**: 한 화면에 정보 1개씩만 표시
5. **스크롤 길이**: 6개 큰 섹션으로 과도하게 긴 페이지
6. **터치 최적화**: 버튼 크기 및 터치 영역 부족

### **개선 목표**
- 히어로 섹션: 100vh → 70vh (모바일)
- 폰트 크기: 반응형으로 모바일 우선 설계
- 섹션 간격: 50% 축소 (모바일)
- 정보 밀도: 한 화면에 2-3개 정보 표시
- 스크롤 길이: 30% 단축
- 터치 영역: 최소 44px × 44px 보장

## 🔧 **구체적 개선 사항**

### **1. 히어로 섹션 개선**
```css
/* 현재 */
.hero { height: 100vh; }
.hero h1 { font-size: 7rem; }

/* 개선 후 */
.hero { 
  height: 70vh;           /* 모바일 */
  @media (md) { height: 100vh; } /* 데스크톱 */
}
.hero h1 { 
  font-size: 1.75rem;     /* 모바일 */
  @media (md) { font-size: 7rem; } /* 데스크톱 */
}
```

### **2. 섹션 간격 최적화**
```css
/* 현재 */
.section { padding: 6rem 0; }

/* 개선 후 */
.section { 
  padding: 2rem 0;       /* 모바일 */
  @media (md) { padding: 6rem 0; } /* 데스크톱 */
}
```

### **3. 카드 레이아웃 개선**
```html
<!-- 현재: 세로 나열 -->
<div class="grid grid-cols-1 md:grid-cols-3 gap-8">

<!-- 개선 후: 수평 스크롤 -->
<div class="flex overflow-x-auto space-x-4 md:grid md:grid-cols-3 md:gap-8">
```

### **4. 터치 영역 최적화**
```css
/* 모든 버튼 최소 크기 보장 */
.btn {
  min-height: 44px;
  min-width: 44px;
  padding: 12px 24px;
}

/* 터치 영역 확장 */
.touch-target {
  position: relative;
}
.touch-target::before {
  content: '';
  position: absolute;
  top: -8px; right: -8px; bottom: -8px; left: -8px;
}
```

## 📱 **모바일 우선 디자인 원칙**

### **정보 계층 구조**
1. **핵심 메시지** (1-2줄)
2. **주요 CTA** (1개 버튼)
3. **핵심 정보** (3-4개 카드)
4. **상세 정보** (접기/펼치기)

### **시각적 우선순위**
1. **크기**: 중요도에 따른 폰트 크기 차등
2. **색상**: 강조 색상 최소화 (1-2개만)
3. **간격**: 관련 정보끼리 그룹화
4. **애니메이션**: 필수 피드백만 유지

### **터치 인터랙션**
1. **버튼**: 최소 44px × 44px
2. **간격**: 터치 요소 간 8px 이상
3. **피드백**: 터치 시 즉시 반응
4. **제스처**: 스와이프, 핀치 지원
